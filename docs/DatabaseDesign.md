# 4.2 Database Design

Database design is the process of creating a structured and organized way to store digital information. It is similar to arranging books in a library, where every item has a proper place, can be easily located, and is protected from loss or damage. In software applications, database design defines how data is stored, connected, retrieved, and maintained.

The main goal of database design is to create a clear blueprint that helps the system store and access information quickly and accurately. It breaks complex information into smaller and manageable parts, identifies relationships between different data entities, and ensures that the data remains consistent, reliable, and secure.

In the Alan Mens Wear ecommerce system, Firebase Firestore is used as the main database. Firestore stores data in collections and documents. The system stores customer profiles, product details, cart items, orders, payment proof, reviews, and generated running codes. Firebase Authentication is used for secure user login and registration, while Firebase Storage may be used for profile photo uploads.

An effective database design improves system performance and user experience. It reduces data duplication, supports faster retrieval, improves accuracy, and makes maintenance easier. It also supports scalability, allowing the application to handle more users, products, orders, and reviews in the future.

## Table 4.2.1 User Schema

Firestore collection: `users/{uid}`

| Field Name | Data Type | Description |
| --- | --- | --- |
| uid | String | Unique user ID generated by Firebase Authentication |
| userCode | String | Custom user code generated by the system, such as `AMWU1` |
| name | String | Customer or admin name |
| email | String | User email address |
| phone | String | User mobile number |
| phoneVerified | Boolean | Indicates whether the phone number is verified |
| address | String | Customer address, added from profile page |
| photoURL | String | User profile photo URL or fallback image data |
| photoStorage | String | Storage type used for the profile photo |
| role | String | User role, either `customer` or `admin` |
| passwordStatus | String | Note indicating password is stored securely in Firebase Auth |
| createdAt | Timestamp | Account creation date |
| updatedAt | Timestamp | Last profile update date |

## Relationships Between Collections

| Relationship | Description |
| --- | --- |
| User to Cart | One user can have many cart items under `carts/{userId}/items` |
| User to Orders | One user can place many orders |
| User to Reviews | One user can submit many reviews |
| Product to Cart | One product can appear in many users' carts |
| Product to Orders | One product can appear in many orders |
| Product to Reviews | One product can have many reviews |
| Order to Payment | Each order contains one embedded payment object |
| Order to Reviews | A delivered order item can be reviewed by the customer |

## Notes

1. Firebase Authentication stores the actual login credentials securely outside Firestore.
2. Firestore stores user profile information, not raw passwords.
3. Payment is not a separate collection in the current project. It is stored inside the order document.
4. Product and payment images are compressed before being saved.
5. Order creation uses a Firestore transaction to reduce product stock and create the order safely.
